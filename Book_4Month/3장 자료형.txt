3장 자료형

정수형
sbyte/1바이트/부호 O
byte/1바이트/부호 X
short/2바이트/부호 O
ushort/2바이트/부호 X
int/unit = 4바이트
long/ulong = 8바이트

실수형
float/4바이트
double/8바이트
decimal/16바이트 (반올림 오차 허용되지 않아 회계에 적합)
* 소수점 있는 연산에는 double이 더 정확하지만, 정밀한 계산이 아니라면 빠른 float가 유리하다.

문자형
char : 유니코드 2바이트
string : 유니코드 문자열

bool
true/false만 담는데 1바이트 크기를 가진다.
* 1바이트인 이유 : 컴퓨터 아키텍처가 주소지정 가능한 가장 작은 메모리 단계가 1byte이기 때문이다.

형변환
부가적 코드 없이 자연스레 형변환 하는것은 암시적. 반대는 명시적 형 변환이라고 한다.
* 자료형이 다르지만, ushort를 char로 변환 가능. (ushort u = 65; char c = (char)u; : c에는 A가 저장)

식별자
임의로 선택해서 이름을 가질 수 있는것으로 변수명, 메서드명, 클래스명 등이 속함.

리터럴
변수의 값을 지정하는 것을 의미한다.

변수
식별자 중 하나로 메모리가 할당되 값을 담는 역할을 수행한다.

값 형식과 참조 형식
윈도우 프로그램은 기본적으로 하나의 스레드를 가진다. 개별 스레드마다 전용으로 사용할 수 있는 저장소가 메모리에 할당되는데,
이를 스택이라고 한다.  반면, 힙은 프로그램에서 필요에 의해 메모리를 사용하겠다고 요청했을 때 사용할 수 있는 저장소이다.

메모리 관리
스택 내 메모리 할당/해제는 C# 컴파일러에 의해 자동으로 관리되지만, 힙은 개발자가 명시적으로 요청한 것이기에 해제 과정이 필요하다.
C#은 관리환경으로 가비지 컬렉터가 힙 영역을 관리한다. (C/C++은 네이티브 환경으로 사용자가 할당/해제를 쌍으로 맞춰야 한다.)
